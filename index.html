Overview

此專案是面試LearnMode學習吧UIUX設計師工作時，面試官出的作業。針對「LearnMode學習吧」這個E-learning平台現階段使用者面臨的痛點與網站架構問題，思考解決方案並重新設計。

LearnMode學習吧平台簡介：學習吧是一個開放與共享的公益教育平臺，主要目的有二：1.透過科技縮減城鄉差距與數位落差。2.針對不同孩子的學習狀況給予最適當的幫助。主要使用者為國小、國中、高中之學生與老師，目前有超過40萬的註冊人數，以及10萬筆以上的數位教育資源(包括：課本、影片、試卷等)。

在課程頁面，我主要依循使用者的兩大需求來設計，分別是「滿足學習需求」與「強化動力」。以下針對每個功能詳細說明：

※以下理論皆有科學/實驗證明。

討論區

  1. 同儕間一起學習、互相討論、能夠有效強化學習動機。
  2. 能夠讓不敢在學校舉手的學生，有機會在討論區中發問
  3. 學生間也可以互相回答對方的問題，教別人就是最有效的內化知識方法，還能夠訓練學生表達能力和同理心。

  1. 讓學生在觀看影片時，遇到問題或課程重點，能夠標記時間點。

BandSearcher

Overview

對於玩音樂的族群來說，通常只能依靠社群網站或論壇等管道來找樂團(或樂手)，但要找到合適的樂團(或樂手)很不容易，主要面臨以下幾個問題：包括無統一平台、檢索不易、徵文格式不一等等。

本作品以使用者的角度切入，實際在FB各大樂手社團，瀏覽了大量徵樂手/樂團的貼文，整理出最常見的使用者需求，並思考解決方案。 最終產出了這個平台，專門讓樂手/樂團們能夠在上面徵人的APP，此APP暫名為BandSearcher。

Tool

  • Adobe XD
  • draw.io

Process

  1. Problem
  2. User Story
  3. Funtional map
  4. Flow Chart
  5. Wireframe
  6. Prototype

Role

Individual project

Problem

  1. 檢索不易：
    找樂手/樂團的條件複雜，需要同時滿足曲風、樂器、年紀、地點等等的條件，但現行社群平台的檢索功能很差，很難找到符合條件的人。
  2. 無統一平台：
    現行沒有一個專門的、整合的徵樂手/樂團平台，樂手/樂團們只能一個一個嘗試不同的FB社團、相關論壇、PTT等，同一篇徵人文要Po多次，非常沒有效率，且造成了資訊的分散與重複，對檢索人來講也是一大困擾。
  3. 徵文格式不一：
    每一篇徵樂手/樂團貼文格式都不一樣，沒有一個統一的格式標準，不但提供的資訊不清楚，且也造成了檢索上的困難。

User Story

樂手：

  1. 想找符合自己條件的樂團(可以自己指定條件，一次列出所有符合自己需求的結果。)
  2. 登上自己的資訊，等待樂團來找。

樂團：

  1. 想找符合自己條件的樂手(可以指定條件，一次列出所有符合自己需求的結果。)
  2. 登上自己的資訊，等待樂手來找。

Funtional map

資訊：

  1. 曲風
  2. 樂器
  3. 年紀
  4. 地點
  5. 性別
  6. 暱稱
  7. 大頭貼

功能：

  1. 關鍵字檢索(找樂團or找樂手)
  2. 進階檢索(條件篩選)
  3. 邀請(非公開)
  4. 留言(公開)
  5. 聊天室(訊息傳輸)
  6. 會員登入功能

Gocha Dog

遊戲連結：Gocha Dog


Overview

這是一場專案競賽：要選擇一個社會問題，提出一個能解決問題的創意提案。
  • Situation：
    我們是一個6人的團隊，要在兩個月內，對流浪狗議題提出一個解決方案，包含完整的發想、規劃、執行。
  • Task：
    經由邏輯樹分析議題後，我們發現「大眾普遍對流浪狗的認識不足」是主要問題，所以我們決定將目標鎖定在「提升潛在領養者對於流浪狗知識的認識」。
  • Action：
    經過腦力激盪後，我們選擇了「遊戲」作為傳遞知識的媒介，製作了流浪狗知識問答小遊戲，並請知名影星和網紅宣傳。
  • Result：
    遊戲在一個月內不重複點擊1265次，轉換率37%。 (完整玩完一次遊戲，並點擊進入流浪狗領養網站，為一次轉換。) 這項專案最終獲得了評審團大獎與最高人氣獎。

Tool

  • 利用Mugeda進行遊戲製作。

Process：

  1. 背景
  2. 決定議題
  3. 議題分析(邏輯樹)
  4. 收斂問題
  5. 訂定解決方案
  6. 製作遊戲
  7. 檢視使用者回饋
  8. 改良

Role

  • 議題分析
  • 訂定解決方案
  • 遊戲製作

利用邏輯樹分析議題+收斂議題

經由邏輯樹分析後，我們將現階段無法解決的項目移除，並整理所剩下來的問題，發現它們是一個共通的原因所導致，那就是「大眾普遍對流浪狗的認識不足」。
※紅色部份表示我們專注要解決的問題。

跳脫框架，選擇以「遊戲」作為解決方案

我們決定將目標鎖定在「提升潛在領養者對於流浪狗知識的認識」後，開始思考要如何有效傳達知識性資訊？常見的方式有宣導文案、懶人包、廣告等等，但知識性資訊通常會讓人無心閱讀，更無法吸引注意，於是我們嘗試跳脫框架，將枯燥的知識以互動遊戲的形式呈現，成功吸引了大家的目光。 我們選用Mugeda這個H5遊戲製作平台，製作了一款類似知識王的知識問答小遊戲。

遊戲細節設計

我們在遊戲中加入了一些巧思，用以提升玩家的遊戲體驗、導正玩家的錯誤觀念、達成我們的轉換目標：

  • 好感度：好感度會因為答題正確或錯誤而提升或降低，來強化玩家的成就感。
  • 最終得分：最終得分能夠讓人為追求滿分而重複遊玩，在遊玩的過程中便能加深印象。
  • 彩蛋：只要玩家沒有獲得滿分，系統便會提示：「獲得滿分就有特別獎勵喔!」(特別獎勵即導向流浪狗領養平台)，促使玩家玩到最後。
  • 糾正：回答錯誤時，我們會立即解說正確答案，及時導正玩家的觀念。
  • 使用者回饋：遊戲的最後有一個回饋系統，玩家能夠做1-5分的評分，並且留下評語/建議，好讓我們能吸收玩家的回饋。

    藉由訊息及電子郵件溝通，接受讀者的反饋，一步一步滿足其資訊需求。有時候讀者可能也不清楚自己的資訊需求，需慢慢引導出來。

    Process

    1.讀者提出問題
    2.第一階段查找
    3.反饋
    4.第二階段查找
    5.反饋
    6.結束
