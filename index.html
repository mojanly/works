About

什麼是鏡像神經元呢？
動物在執行或是觀察到其它個體動作時，大腦中有某些神經元的活性會增加， 在腦中模仿、重現該動作或情緒，也就是所謂的「感同身受」，這些神經元就被稱為鏡像神經元。

本網站以鏡像神經元為名，期望能秉持同理心，以使用者的角度， 設計出符合他們需求的產品。 未來這裡會持續更新有關UI/UX相關的學習筆記、學習資源，以及介面設計、平面設計、使用者研究的相關作品， 敬請期待！

1000樓電梯之控制介面設計

作品簡介：

為一個多達1000層樓層的電梯，設計一個控制電梯的介面。 使用者可以不用操作電梯本身的操作面板，使用隨身裝置即可控制電梯並且得知電梯目前的狀態資訊。 作品考慮到了電梯的大小、移動速度、移動時間、搭乘人數、動線、使用者心理、使用流程等面向， 來設計一款能符合使用者資訊需求與安全感的介面。

使用工具：

利用Adobe XD進行介面製作與互動設計。

作品設計流程：

  1. Background
  2. User Story
  3. Flow Chart
  4. Mockup
  5. Prototype

BandFinder(製作中)

作品簡介：

對於玩音樂的族群來說，要找到合適的樂團或是樂手一直是很大的問題。社群網站/論壇是目前找樂手/樂團的重要管道，但這些管道有以下幾個問題：

  1. 找樂手/樂團的條件複雜，需要同時滿足曲風、樂器、年紀、地點等等的條件，但現行社群平台的檢索功能很差，很難找到符合條件的人。
  2. 現行沒有一個專門的、整合的徵樂手/樂團平台，樂手/樂團們只能一個一個嘗試不同的FB社團、相關論壇、PTT等，同一篇徵人文要Po多次，非常沒有效率，且造成了資訊的分散與重複，對檢索人來講也是一大困擾。
  3. 每一篇徵樂手/樂團貼文格式都不一樣，沒有一個統一的格式標準，不但提供的資訊不清楚，且也造成了檢索上的困難。

因此本作品提供了一個專門徵樂手/樂團的APP，暫名為BandFinder，期望能解決上述所有的問題。本作品以使用者的角度切入，實際去FB各大社團，瀏覽了大量徵樂手/樂團的貼文，整理出幾個最常見的使用者需求，再思考解決方案。

使用工具：

利用Adobe XD進行介面製作與互動設計。

作品設計流程：

  1. Background
  2. User Story
  3. Funtional map
  4. Flow Chart
  5. Wireframe
  6. Clickable Wireframe
  7. Prototype

Gocha Dog

遊戲連結：

遊戲連結

作品簡介：

此為資訊種子TUV專案之作品，在兩個月的時間內，選擇一個社會議題，作為目標。
我們都曾看過被棄養的流浪狗，但總是苦於心有餘而力不足，而沒有辦法改變什麼。 在這個專案中，我們終於有機會為流浪狗的問題盡一份力，哪怕是只有一點點，我們也希望能為這個世界做出一些改變。 我們決定以流浪狗議題做為我們要解決的問題，進而製作了這個網頁互動小遊戲，希望能以趣味的形式，將這個流浪狗的議題與常識，傳達給更多人。 最後，我們的專案獲得了評審團大獎和人氣獎的殊榮。

使用工具：

利用Mugeda進行遊戲製作。

專案發想流程流程：

  1. 決定議題
  2. 問題分析(邏輯樹)
  3. 收斂問題
  4. 決定解決方案
  5. 製作遊戲
  6. 檢視使用者回饋
  7. 改良

分類法

報告簡介：

分類法為資訊架構的一環，此報告主要是將"層面分類法"的概念應用在電子商務網站的商品分類，以使用者的角度去思考，什麼樣的分類方式是最適合使用者的。

負責工作：

主題發想、 分類層面發想、 分類表實作。

報告製作流程：

1.決定以電腦作為主題
2.列出使用者在選購電腦時最常作為篩選的條件
3.決定各條件的分類細項，思考如何怎樣的區塊分隔能符合不同族群的使用者
4.分類表實作

資訊需求研究

報告簡介：

藉由訊息及電子郵件溝通，接受讀者的反饋，一步一步滿足其資訊需求。有時候讀者可能也不清楚自己的資訊需求，需慢慢引導出來。

報告設計流程：
