BandSearcher

Overview

對於玩音樂的族群來說，通常只能依靠社群網站或論壇等管道來找樂團(或樂手)，但要找到合適的樂團(或樂手)很不容易，主要面臨以下幾個問題：包括無統一平台、檢索不易、徵文格式不一等等。

本作品以使用者的角度切入，實際在FB各大樂手社團，瀏覽了大量徵樂手/樂團的貼文，整理出最常見的使用者需求，並思考解決方案。
最終產出了這個平台，專門讓樂手/樂團們能夠在上面徵人的APP，此APP暫名為BandSearcher。

Tool

  • Adobe XD
  • draw.io

Process

  1. Problem
  2. User Story
  3. Funtional map
  4. Flow Chart
  5. Wireframe
  6. Prototype

Role

Individual project

Problem

  1. 檢索不易：找樂手/樂團的條件複雜，需要同時滿足曲風、樂器、年紀、地點等等的條件，但現行社群平台的檢索功能很差，很難找到符合條件的人。
  2. 無統一平台：現行沒有一個專門的、整合的徵樂手/樂團平台，樂手/樂團們只能一個一個嘗試不同的FB社團、相關論壇、PTT等，同一篇徵人文要Po多次，非常沒有效率，且造成了資訊的分散與重複，對檢索人來講也是一大困擾。
  3. 徵文格式不一：每一篇徵樂手/樂團貼文格式都不一樣，沒有一個統一的格式標準，不但提供的資訊不清楚，且也造成了檢索上的困難。

User Story

樂手：

  1. 想找符合自己條件的樂團(可以自己指定條件，一次列出所有符合自己需求的結果。)
  2. 登上自己的資訊，等待樂團來找。

樂團：

  1. 想找符合自己條件的樂手(可以指定條件，一次列出所有符合自己需求的結果。)
  2. 登上自己的資訊，等待樂手來找。

Funtional map

資訊：

  1. 曲風
  2. 樂器
  3. 年紀
  4. 地點
  5. 性別
  6. 暱稱
  7. 大頭貼

功能：

  1. 關鍵字檢索(找樂團or找樂手)
  2. 進階檢索(條件篩選)
  3. 邀請(非公開)
  4. 留言(公開)
  5. 聊天室(訊息傳輸)
  6. 會員登入功能

Gocha Dog

遊戲連結：Gocha Dog


Overview

  • Situation：我們是一個6人的團隊，要在兩個月內，對流浪狗議題提出一個解決方案，包含完整的發想、規劃、執行。
  • Task：經由邏輯樹分析議題後，我們發現「大眾普遍對流浪狗的認識不足」是主要問題，所以我們決定將目標鎖定在「提升潛在領養者對於流浪狗知識的認識」。
  • Action：經過腦力激盪後，我們選擇了「遊戲」作為傳遞知識的媒介，製作了流浪狗知識問答小遊戲，並請知名影星和網紅宣傳。
  • Result：遊戲在一個月內不重複點擊1265次，轉換率37%。 (完整玩完一次遊戲，並點擊進入流浪狗領養網站，為一次轉換。) 這項專案最終獲得了評審團大獎與最高人氣獎。

Tool

  • 利用Mugeda進行遊戲製作。

Process：

  1. 背景
  2. 決定議題
  3. 問題分析(邏輯樹)
  4. 收斂問題
  5. 決定解決方案
  6. 製作遊戲
  7. 檢視使用者回饋
  8. 改良

Role

  • 議題分析
  • 遊戲製作

    資訊架構之分類法

    Overview

    分類法為資訊架構的一環，此報告主要是將"層面分類法"的概念應用在電子商務網站的商品分類，以使用者的角度去思考，什麼樣的分類方式是最適合使用者的。

    Role

    主題發想、 分類層面發想、 分類表實作。

    Process

    1.決定以電腦作為主題
    2.列出使用者在選購電腦時最常作為篩選的條件
    3.決定各條件的分類細項，思考如何怎樣的區塊分隔能符合不同族群的使用者
    4.分類表實作
